# Project Structure

The bigger your project is the better your structure should be.

For this particular project I've structured it in a way that I find good.

Complexity should also match the project size.

## NodeNote.js file structure

Under here I will show my project structure at the time of writing this - 11/03/2026T20:03

```
DocumentationSite
├── node_modules/
├── public/
├── tailwind/
├── util/
├── .gitignore
├── app.js
├── eslint.config.mjs
├── LICENSE
├── package-lock.json
├── package.json
└── README.md
```

As shown above my root folder contains mostly Node.js specific files and folders.

I have created the **util** folder to hold all the scripts that will only be running on the server and not on the client

```
util/
└── notesUtil.js
```

The **tailwind** folder contains all files that are used by tailwind

```
tailwind/
├── header.css
├── input.css
├── main.css
├── mainSection.css
└── sidebar.css
```
