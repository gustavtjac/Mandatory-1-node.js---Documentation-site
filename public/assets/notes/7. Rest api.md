# Rest Api - Representational State Transfer API

Rest is basically a set of rules and conventions to follow while building a web API's.

## HTTP methods

There are five main http methods that we have gone over this semester an they are the following

1. GET - Represents methods that gather data
2. POST - Represents methods that creates new data
3. PUT - Represents methods that updates a whole row / object
4. PATCH - Represents methods that updates parts of row / object
5. DELETE - Represents methods that deletes data

## Rest api standards and conventions

### Endpoint naming

Many people choose to add the prefix /api to their endpoints, like this:

```url
/api/examples
```

This is done to keep seperation of rest api endpoints and frontend site serving endpoints

It's also mandatory to have whatever object your calling in plural inside your url. I will show an example with an api for a user.

```url
/api/users
```

### Order of endpoints

inside our backend code its also a rule that we should have our endpoint-methods in the same order as the list on top of the site.

First all GET endpoints then POST then PUT then PATCH then DELETE.

## Example of a Rest api design

```rest
GET    /users          → get all users
GET    /users/:id      → get user with id
POST   /users          → create a new user
PUT    /users/:id      → update user with id
DELETE /users/:id      → delete user with id
```
