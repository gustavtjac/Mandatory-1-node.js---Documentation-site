# CommonJS vs ESModule

When you create a new Node.js project it will be set to CommonJS by default.

However that is not necessarily a good thing. The new standard for Node.js and JavaScript is ESModule.

CommonJS is a Node.js specific thing, which means that you will end up having a different syntax for frontend and backend.

It will also be harder to reuse code, for example if you need to sanitize some input in both the frontend and backend, then the code needs to be written in two different ways.

## How to switch to ESModule

```json
//package.json
{
    "type": "module"
}
```

All you need to do to use ESModule is to set the key "type"'s value to "module".

## Differences in code

### Import / Require - Export / module.exports

Import and export syntax are very different.

**ESModule**

```javascript
import express from 'express';
```

**CommonJS**

```javascript
const express = require('express');
```

To export in ESModule you use the `export` keyword.
In CommonJS you use `module.exports`.

**ESModule**

```javascript
export function myFunction() {}
```

**CommonJS**

```javascript
module.exports = { myFunction };
```
