# Content Delivery Network vs serving content locally

Serving a script or styling through a CDN vs locally each has its pros and cons.

## Fast for users all over the world

CDNs often have servers all over the world. For example, this means that a user in Korea could probably get the files from a server in Asia instead of from my home server in Denmark, which would decrease load times for them.

## Less load on server

By using a CDN we basically outsource all of that load to the CDN, which means that our server has to do less work.

## Caching

If a user has browsed other sites that use some of the same scripts from the same CDN, they are probably cached and won't have to be loaded again.

## Does not work offline

If you don't have an internet connection your script or styling won't load because you first need to download from the CDN using internet.

Serving it from your own server will work perfectly offline as long as you're on the same network.

## Depending on a third party

If you use a CDN and they out of nowhere change their code to work in a different way, your site will break without you being able to do anything about it.

Serving it locally will allow you to use a specific version and it will only ever change if you make it.
