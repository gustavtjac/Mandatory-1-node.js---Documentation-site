# Anatomy of a URL

A URL is made up of several parts.

```
https://www.google.com/search?q=cool+search&newwindow
```

Let's use the URL above as an example.

| Part                | Example                           | Description                                            |
| ------------------- | --------------------------------- | ------------------------------------------------------ |
| Protocol            | `https`                           | Defines the protocol used to transfer data             |
| Protocol identifier | `://`                             | Separator between the protocol and the rest of the URL |
| Subdomain           | `www`                             | A prefix to the main domain                            |
| Domain              | `google`                          | The name of the domain                                 |
| Top-level domain    | `.com`                            | The category or country of the domain                  |
| Endpoint            | `/search?q=cool+search&newwindow` | Points to a specific resource or page                  |
| Query string        | `?q=cool+search&newwindow`        | Key-value pairs passed to the server                   |

## Path Variable

A path variable is a way of sending content dynamically based on the path.

```
/user/1
```

In the example above **1** is the path variable, and should return the user with the id of **1**.

## Query String

A query string starts with **?** and contains key-value pairs separated by **&**:

```
?q=cool+search&newwindow
```

| Key         | Value               |
| ----------- | ------------------- |
| `q`         | `cool+search`       |
| `newwindow` | _(no value, flag?)_ |
