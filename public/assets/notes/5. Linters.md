# Linters & Linting

## What is a linter

A linter is a tool that does some static code analysis.
Depending on how you set it up and which linter you are using it will catch diffrent things.
Generally linters look for unformatted code, unused variables, unsafe dependencies etc..

They can really help maintain a higher code quality, especially if you are like me :)

## What are some examples of linters?

Here are some examples of JavaScript linters. I Personally use ESLint.

- ESLint
- JSHint
- StandardJS
- Prettier

## How to install ESLint

Linters are development dependencies. We are going to download it using npm and save it as a dev-dependency and then run its init function to get it setup

```bash
npm install --save-dev eslint
npx eslint --init
```

## How to use ESLINT

After installing ESLint we can run it by writing the following

```bash
#Check whole project
npx eslint

#Check specific JavaScript File
npx eslint file.js
```

If your code doesnt pass the linting it will print the problems in your console like these i just got

```bash
  13:14  error  'err' is defined but never used  no-unused-vars
```

It tells you the file, line and char in which the problem occured.
